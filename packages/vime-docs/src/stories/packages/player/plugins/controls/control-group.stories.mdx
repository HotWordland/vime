# ControlGroup `Component`

[View Source](https://github.com/vime-js/vime/blob/master/packages/vime-player/src/plugins/controls/ControlGroup.svelte)

Manages rendering and laying out a group of controls. The layout is managed via 
[CSS Flexbox](https://css-tricks.com/snippets/css/a-guide-to-flexbox/) properties.

## Props

### `controls: Control[]`

**Defaut:** `[]`

The collection of controls (see <LinkTo kind="packages-player-plugins-controls-control-control--page">Controls/Control</LinkTo>) 
to be grouped together and rendered.

### `isEnabled: boolean`

**Defaut:** `true`

Whether the group should be rendered into the DOM or not.

### `isActive: boolean`

**Defaut:** `false`

Whether the group is visible or not.

### `shouldFill: boolean`

**Defaut:** `false`

Whether the group should take up all available space in its container by setting `flex: 1` on the 
root component element.

### `flow: string|null`

**Defaut:** `null`

Sets the `flex-flow` CSS property of the group (see [W3 Schools - flex-flow property](https://www.w3schools.com/cssref/css3_pr_flex-flow.asp)).
By default the `flex-flow` is `row wrap`.

### `position: string|null`

**Defaut:** `null`

Sets the `align-items` and `justify-content` CSS properties of the group. This property is written 
as `(align-items):(justify-content)`. For example, `flex-center:flex-start`.

## Methods

### `getEl(): HTMLElement`

The root HTML element of the component.

### `hasControls(): boolean`

Whether the group has any controls or not.

### `reset()`

Reset all props to their default values.

### `getInstances(): { [id]: Control }`

The rendered `Control` instances.
